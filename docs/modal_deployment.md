# توثيق نشر نماذج التضمين (Embedding Models) على Modal.com

لتخفيف العبء عن جهازك المحلي (CPU/RAM) وتسريع عملية تحويل النصوص إلى متجهات (Vectors)، قمنا بفصل الـ Embedding Model ورفعه كخدمة مستقلة (Microservice) على منصة Modal.com.

## الخطوات:

### 1. إعداد بيئة Modal
أولاً، يجب أن تمتلك حساباً في [Modal.com](https://modal.com/) ثم تقوم بتثبيت المكتبة والمصادقة:
```bash
pip install modal
modal setup
```

### 2. الكود السحابي (`modal_embed_api.py`)
تم كتابة ملف `modal_embed_api.py` في الجذر الرئيسي للمشروع. 
هذا الملف يطلب من Modal تجهيز سيرفر يحتوي على بطاقة رسومية (GPU) من نوع `T4` لضمان أقصى سرعة في استرجاع البيانات، وتثبيت `FlagEmbedding` و `FastAPI`، ثم يقوم بتحميل الموديل `BAAI/bge-m3` (4.5 جيجا) مباشرة من سيرفرات HuggingFace (وليس من الإنترنت الخاص بك).
ملاحظات برمجية هامة داخل الكود:
- نستخدم المزخرف `@modal.fastapi_endpoint(method="POST")` لإنشاء الـ API.
- نستخدم المزخرف `@app.cls(image=image, gpu="T4")` لضمان تخصيص معالج رسومي للسيرفر بدلاً من الـ CPU.
- نقوم بتحويل كل الأوزان (Weights) التي يخرجها الموديل من صيغة `numpy.float16` إلى `float` العادي الخاص بـ Python لكي يعمل `FastAPI` بسلاسة ولا يحدث خطأ في إرسال الـ JSON (Serialization Error).

### 3. رفع الخدمة (Deployment)
تأكد من أنك داخل مسار المشروع الصحيح:
```bash
cd "Graduation Project Zad\Zad-AI"
```
ثم قم بتشغيل أمر النشر:
```bash
modal deploy modal_embed_api.py
```
سيقوم هذا الأمر ببناء الصورة السحابية، وسيعيد لك رابطاً جاهزاً (URL) يبدأ بـ `https://`. احتفظ بهذا الرابط.

### 4. ربط Zad-AI بالرابط الجديد عبر `.env`
بعد الحصول على الرابط السحابي، قم بإضافته إلى ملف البيئة `.env` كالتالي:
```env
MODAL_EMBEDDING_URL=https://your-app-url.modal.run
```

سيقوم كود `bge_m3_model.py` تلقائياً باكتشاف هذا الرابط واستخدامه بدلاً من تحميل النموذج محلياً. إذا تم مسح الرابط من ملف `.env`، سيقوم الكود أوتوماتيكياً بالعودة للطريقة القديمة (تحميل النموذج على الرامات المحلية) كخطة بديلة (Fallback).
