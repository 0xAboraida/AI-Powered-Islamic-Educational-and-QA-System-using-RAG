# خطة تنفيذ ميزة الإدخال الصوتي (Voice Input) 🎙️

إضافة ميزة التسجيل الصوتي للمحادثات هي إضافة ممتازة وتجعل الـ Chatbot أكثر تفاعلاً. لتحقيق ذلك بأعلى كفاءة في مشروع Zad-AI، قمت بإعداد هذه الخطة المعمارية.

## الهدف
السماح للمستخدم بإرسال ملف صوتي (Voice Note) ليتم تحويله إلى نص (Speech-to-Text) بدقة عالية باللغة العربية، ثم استخدامه للبحث وتوليد الإجابة.

## User Review Required
> [!IMPORTANT]
> **اختيار تقنية تحويل الصوت إلى نص (STT):**
> نظراً لأن السيرفر الخاص بك سيعمل عبر Docker (غالباً بدون كارت شاشة GPU قوي للمشروع ككل)، فإن تشغيل نموذج Whisper محلياً سيجعل حجم الـ Image ضخماً جداً وسيكون بطيئاً جداً.
> **لذلك، أقترح بشدة استخدام واجهة برمجة التطبيقات (API) الخاصة بـ Groq (نموذج Whisper V3) أو OpenAI.**
> Groq سريع جداً (شبه لحظي) ويدعم العربية بامتياز، ونحن نمتلك مفتاح Groq بالفعل في المشروع. هل توافق على استخدام Groq API لتحويل الصوت؟

> [!WARNING]
> **التصميم المعماري للـ Endpoint:**
> أقترح ألا نقوم بدمج (تحويل الصوت + توليد الإجابة) في مسار واحد. بل أقترح إنشاء مسار (Endpoint) منفصل اسمه `/api/v1/chat/transcribe`.
> - **السبب:** في التطبيقات الشرعية، المصطلحات دقيقة جداً. إذا قام النظام بتحويل الصوت إلى نص، يجب أن نعيده للـ Frontend (تطبيق Flutter) ليضعه في مربع النص، لكي يراه المستخدم ويتأكد أنه صحيح قبل إرساله، تماماً مثلما يحدث في تطبيقات الذكاء الاصطناعي الحديثة.
> هل توافق على هذا التصميم، أم تفضل أن يرسل المستخدم الصوت ونرد عليه بالإجابة النهائية مباشرة؟
