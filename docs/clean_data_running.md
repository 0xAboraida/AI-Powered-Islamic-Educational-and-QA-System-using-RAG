هذا السكربت (`01_clean_data.py`) مصمم ليكون أداة معالجة مرنة للغاية (Production Utility) لتنظيف النصوص العربية. 

وظيفتها الأساسية هي قراءة نصوص الكتب الإسلامية الخام (الموجودة في `data/02_extracted`) وتمريرها على كلاس `ArabicTextCleaner` لتنظيفها من الرموز الزائدة والمعايرة، ثم حفظ الناتج بنفس الهيكل التنظيمي والمجلدات الفرعية داخل `data/03_cleaned`.

بفضل نظام مسارات بايثون الديناميكي الجديد، يمكنك تشغيل هذا الملف من أي مسار (Terminal CWD). إليك طرق تشغيل واستخدام الكود بالتفصيل:

---

### 1. تشغيل السكربت على كل شيء (تنظيف كامل البيانات)
إذا أردت تنظيف كل الكتب الموجودة في مجلد `data/02_extracted` بجميع العلوم والمذاهب دفعة واحدة:
* **من المجلد الرئيسي للمشروع (`Zad-AI`):**
  ```bash
  python services/ai_rag_engine/app/scripts/01_clean_data.py
  ```
* **أو إذا دخلت داخل مجلد الـ `scripts`:**
  ```bash
  cd services/ai_rag_engine/app/scripts
  python 01_clean_data.py
  ```

---

### 2. تشغيل السكربت على "علم/مجال" محدد (Domain)
إذا كنت تريد تنظيف كتب العقيدة فقط، أو الفقه فقط، مرر معامل `--domain`:
```bash
python 01_clean_data.py --domain "1- Fiqh"
# أو
python 01_clean_data.py --domain "4- Tafseer"
```
*السكربت سيبحث داخل `data/02_extracted/1- Fiqh` فقط ويحفظ المخرجات في `data/03_cleaned/1- Fiqh`.*

---

### 3. تشغيل السكربت على "مذهب" محدد (Madhhab)
إذا أردت معالجة مذهب معين فقط (مثل المذهب الحنفي) تحت الفقه:
```bash
python 01_clean_data.py --domain "1- Fiqh" --madhhab "hanafi"
```
*سيقوم بمعالجة الملفات في `data/02_extracted/1- Fiqh/hanafi` فقط.*

---

### 4. تشغيل السكربت على "كتاب محدد" عبر الـ ID
إذا أردت تنظيف كتاب واحد فقط دون البقية، مرر الـ ID الخاص به (مثلاً `1067` لكتاب الهداية):
```bash
python 01_clean_data.py --book-id "1067"
```
*سيقوم بالبحث في كل المجلدات عن كتاب يبدأ اسمه بـ `book_1067` ويعالجه بمفرده.*

---

### 5. تشغيل السكربت على "ملف محدد" مباشرة بالمسار
إذا كان لديك ملف JSON في مسار خارجي أو محدد وتريد تشغيله مباشرة:
```bash
python 01_clean_data.py --file "path/to/your/book.json"
```

---

### ⚙️ خيارات متقدمة لتغيير مجلدات المدخلات والمخرجات
بشكل افتراضي، يقرأ السكربت من `data/02_extracted` ويكتب في `data/03_cleaned`. لكن يمكنك تغيير هذه المسارات تماماً عبر:
```bash
python 01_clean_data.py --input-dir "custom/input/path" --output-dir "custom/output/path"
```

### كيف يعمل الكود داخلياً؟
1. **`discover_files`**: تبحث بشكل شجري (`glob("**/*.json")`) في المجلد المحدد وتطبق الفلاتر التي قمت بتمريرها في الـ CLI.
2. **`process_file`**: تقوم بفتح الملف، تشغيل الـ Cleaner على حقل `content` في كل كائن، ثم الحفظ.
3. **`relative_to`**: تأخذ المسار النسبي للملف المدخل (مثلاً `1- Fiqh/hanafi/book.json`) وتنشئ مجلدات مطابقة تماماً له في المخرجات للحفاظ على نظافة وترتيب مشروعك.
